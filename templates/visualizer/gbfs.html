Greedy Best First Search

What is it?

and how does it work?

With that negative ring to its name you might be wondering what it does to deserve it. Technically the Greedy Best algorithm is similar to the A* algorithm in a lot of ways. In fact there are just 2 technical points that distinguish these algorithms from one another. (Please read the A* description if you haven’t yet) The A* algorithm keeps in mind the distance already travelled from the starting point where as the GBFS lives completely in the moment so to speak. It doesn’t care about the previous data, all it cares about is how it moves forward from its current position. We all have a colleague like this, don’t we? Hence the greedy best search continuously keeps moving towards the destination node without thinking if it has made any mistakes getting there.

Now due to this greedy approach, this algorithm might always find a way to the destination but much like in real-life it might sometimes ends up taking a longer route. Because if we are being honest, life isn’t just all about ambition.

Just like in A*, Greedy Best First works by creating a lowest cost tree-diagram for every node it visits and then continues with the node that has the least heuristic value or in simpler words is closer to the destination. This process is repeated till the algorithm reaches its destination.

GBFS uses 2 functions f(n) and h(n) unlike A*, and here f(n)=h(n). The f(n) is the total estimated cost from the starting point to destination through n, and h(n) is the estimated cost from n to the destination.
